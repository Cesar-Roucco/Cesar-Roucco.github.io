Weather Satellite Receiver

Fall 2019

For this project, I designed, manufactured and tested every component of a Weather Satellite Receiver System as shown by the diagram above. The receiver consists of a quadrifilar antenna that picks up the receiving signal from a NOAA-19 satellite orbiting the earth. Then the Low Noise Amplifier (LNA) amplifies it while keeping the noise figure of the system low. Then the Band-Pass Filter (BPF) filters the high-frequency RF signal to the expect FM band. Using a local oscillator circuit and a mixer, the RF signal gets mixed down to an intermediate IF signal. Finally, this signal is fed to the FM demodulator in order to retrieve an audio file of the data. Finally, the audio file was pre-processed and converted to an image file using a Weather-to-Image Software. The decoded picture can be seen above.

Semi-Autonomous UGV

Fall 2018 - Spring 2019

In this project I collaborated in an interdisciplinary group of 5 engineering students to design a 3-D printed UGV capable of switching manual and autonomous operations for unknown space exploration. The platform used is ArduPilot open source hardware, along with a NVIDIA Jetson TX2 companion computer handling the autonomous operations. The system is built on the Robot Operating System (ROS) software framework and communicates using a MAVLink protocol. ROS allows for various sensor integration onto the rover using LIDAR, accelerometer and a camera. With the LIDAR we can detect obstacles and use SLAM algorithms to generate maps. Using these maps we can navigate through an explored space. The camera is used for visual odometry to improve the locational accuracy.

Hand-held Cube Game

Fall 2018

For this project, I built a Hand-held video game called "Cube Game". This game was built using a TI TM4C123G microcontroller and a TI Educational BoosterPack MKII, which incorporates an analog joystick, RGB LED, microphone, buzzer, color LCD display, buttons, accelerometer, etc. The system is built using a Real-Time Operating System (RTOS), which was designed by me during the course of the semester for my Advanced Embedded Systems course. The game uses the joystick, both buttons, and the accelerometer for physical interaction, and an LCD screen for visualization. Other features include a menu, instructions, chill mode, and accelerometer data used to enter the bonus mode by shaking the console. RISC-V Processor

Spring 2018

This project was the final project for my Computer Architecture and Design class. I built a RISC-V processor using VHDL and designed each of the 5 stages in the pipeline, as well as the register file, register tracker to deal with data hazards, and memory arbiter to deal with structural hazards. The first stage of the processor fetches the instructions from memory and sends it to the second stage. The second stage then decodes it into a specific action and grabs data from the register file for the ALU to perform an operation in the third stage. The third stage then executes the instruction and the output is sent to the fourth stage, where the processor could store data into memory if necessary or send it to the fifth stage to write it back into the register file. The entire processor was succesfully synthesized using Precision Synthesis tool from Mentor Graphics and was also tested by running binary code that divides one number from another and stores the result in one of the registers.

LockBox

Spring 2018

The Lock-Box is a project inspired by the need for some students to buy really expensive textbooks for a class. This project focused on designing a smart book-storing and book-sharing system that would allow multiple users to store a textbook in the box and the box would record whether a book was present at the moment or not. The user would need to register with the system and obtain their unique passcode to the box. This user can then add other users to the box in order to start sharing books. The box uses pressure sensors made from velostat to determine whether an object is placed on the shelf of the box (detecting a book) and detects whether a shelf is empty or full, updating a database and a website that is used to provide the user with system feedback. The system also features an LCD display that allows the user to see its passcode input. The entire circuitry and the Arduino are stored in the back of the lockbox and secured using a locker lock. This project was the final project for my Integrated Systems Design class.

Line-Following Delivery Robot

Fall 2017

This project was an extension to the Follow-the-Leader Robot. This project was changed a bit by making it a delivery robot that can go to a certain destination by taking the shortest path. The shortest path is calculated using Dijkstra's shortest path algorithm, taking into account the nodes and each of their weights. We also created a very simple website that allowed us to enter the destination we wanted our robot to go to, so that it can then be saved in a database and be accessed by the robot when it is doing a delivery. The robot accesses the information that was saved in a database, calculates the shortest path to that destination and produces a sequence of commands for the robot to take. We stored the list of destinations in a database and then recorded the ones that had been completed and the ones that still hadn't completed using a CC3000 Wifi chip in the Arduino Uno. The database kept being updated as new destinations were entered as well as when the robot completed a delivery.

Accelerometer Controlled LEDs

Fall 2017

This project was the final project for my Intro to Embedded System Design class. It consists of a board with an accelerometer that measures the tilt of the system from the x-y plane, as well as the direction it is facing in order to control an array of LEDs that are positioned to represent cardinal directions using a TI MSP430 microcontroller. In the first picture, the system is leveled and therefore all LEDs shine at the same brightness. However, when the system starts to tilt, the LED closest to the ground exhibits a brighter intensity and the ones farther from the ground begin to decrease in intensity. In second picture, the LED closest to the ground is in the East position so it shines brighter than the others. In the last picture, we can see the system rotated once again, and now the LED that is closest to the ground is the one facing north, so it shines brighter than the others.

Electrocardiogram Sensor PCB

Fall 2017

This project uses basic electrical engineering concepts to design an electrocardiogram (EKG) printed circuit board (PCB). The board takes 3 very low voltage signals from the body as inputs, two different signals come from the wrists and the third one is a reference from either the ankle or elbow. The board takes the first 2 signals through an instrumentation amplifier and amplifies the voltage difference between them while isolating the desire signal from the heart. However, because other unwanted frequency signals will be present, a 4th order Butterworth filter is used to eliminate high frequencies. Each component of the PCB was assembled and tested individually and then assembled. The final design for this project was powered by an external battery. After the testing stage, I was able to attach the electrodes to the body and see a heart rate in the monitor.

Line-Following Follow-the-Leader Robot

Spring 2017

In this project, a group of 3 engineering students and I designed and built a line-following robot from scratch using laser cutting for the chassis, an Arduino Uno microcontroller and various sensors and actuators. The line-following robot follows a leader robot that was designed by the instructor of the class and maintains a fixed distance behind it using an ultrasonic distance sensor. It uses reflectance sensors to follow a line (black electrical tape on the floor). At intersections, the leader robot sends an IR signal to the follower robot which has an IR demodulator on board that decodes the message and decides to either make a left turn, right turn or continue moving forward. A motor driver is in charge of selecting the direction and setting the PWM values for the motors. Finally, the entire robot is powered using a 9V battery.

Audio Spectrum Visualizer

Spring 2017

This project uses basic electrical engineering concepts to design an audio visualizer that takes in an audio signal as input and visualizes the bass and treble of a song using a green and red LED respectively. The board uses a low-pass filter for the bass, and a high pass filter for the treble both in the Sallen-Key configuration. After the audio signal gets filtered out, a voltage divider then determines the gate voltage for the MOSFET that sets the current for the respective LED. The LED's brightness is changed using a MOSFET low-side driver, and a voltage follower is used to protect the gate of the MOSFET. Finally, the entire board was powered using a 9V battery.
